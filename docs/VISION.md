## 一句话

**CrabCrush 是面向中国用户的本地优先、风险可控的个人 AI 执行助手。**

它不是单纯的聊天机器人，而是让用户在自己的设备上，借助本地 Web 控制面和常用聊天入口，调度一个真正能安全执行任务的 AI。

## 为什么做这个

中国用户在 AI 助手这件事上长期处于一个尴尬位置：

1. 国际项目往往渠道、模型、部署环境都更偏欧美生态
2. 国内很多产品是封闭平台，数据和控制权不在用户手里
3. 想自己搭一套可用、可控、能扩展的 AI 助手，门槛仍然偏高

CrabCrush 想解决的是：**让中国用户也能拥有一个真正属于自己、运行在自己环境里、边界清楚的 AI 执行助手。**

## 目标用户

| 用户类型 | 典型场景 | 优先级 |
|---------|---------|--------|
| 技术人员 | 编程辅助、知识管理、自动化脚本 | P0 |
| 小团队 / 创业者 | 团队助手、内部知识库、流程自动化 | P0 |
| 个人效率用户 | 信息整理、浏览器操作、重复任务自动化 | P1 |
| 企业 IT 部门 | 本地部署、数据可控的内部助手 | P2 |

## 核心定位

CrabCrush 不想做下面这些东西：

- 不是 LLMOps 平台
- 不是单一渠道聊天机器人
- 不是云端 SaaS 助手
- 不是“大而全”的功能集合

它的差异化定位是：

1. **本地优先**：历史、工作区、审计和执行环境掌握在用户自己手里
2. **风险可控**：高风险操作必须确认、可审计、可中断
3. **中文原生**：优先服务中国用户的渠道、模型和使用习惯
4. **执行优先**：真正有壁垒的是“安全地能干活”，不是多接几个入口

## 战略优先级

当前阶段的优先级顺序是：

1. **可控执行底座**：权限、确认、授权作用域、执行预览、审计回放
2. **高价值工具闭环**：浏览器、文件、数据库、受限执行
3. **任务体验**：本地 Web 控制面的任务确认、摘要和管理能力
4. **渠道扩展**：飞书、企业微信等
5. **更远期能力**：知识、语音、多模态、生态建设

这也是 `DEC-035` 的核心收束方向。

## 成功标准

### 已完成的 V1

V1 已于 2026-02-13 发布，完成：

- WebChat + 钉钉双渠道纯对话
- DeepSeek + 通义千问双模型
- 本地部署 + 流式输出 + 多轮上下文

### V2+ 期望结果

- 用户可在本地 Web 控制面发起多步任务，并清楚知道 AI 将要做什么
- 至少形成 3 类稳定、高价值、可控的执行工具闭环
- 至少拥有 1 个强控制面入口和 2 个稳定聊天入口
- 在保持边界清楚的前提下，逐步扩展知识、语音和本地模型能力
